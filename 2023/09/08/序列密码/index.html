# 序列密码概述

序列密码的加密用一个随机序列 (密钥流) 与明文序列按位加密产生密文，用同一随机序列与密文序列叠加来恢复明文。简单来说，序列密码是指明文消息按字符 (如二元数字) 逐字符地加密的一类密码算法，序列密码加密 / 解密过程如图所示：(序列密码的体制模型)

其中，K 是系统密钥源，也称初始密钥，它通过安全信道进行传输，通信双方在拥有相同的密钥 k 的前提下，能够通过密钥序列产生器生成相同的密钥序列：k0k1...ki...kn-1。令 m0m1...mi...mn-1，mi∈M 是待加密的消息序列。密文序列 c=c0c1...ci...cn-1=Ek0 (m0) Ek1 (m1)...Eki (mi)...Ekn-1 (mn-1)，ci∈C，若 ci=Eki (mi)=mi⊕ki，则称这类为加法序列密码。序列密码通常可以被划分为同步序列密码和自同步序列密码两大类。

# 分组密码和序列密码区别（对称密码）

①分组密码把明文分成相对较大的块，对于每块使用相同的加密函数进行处理，因此，纯分组密码是无记忆的。
②序列密码处理的明文长度可以小到 1 比特，序列密码 (有时也被称作状态密码) 往往是有记忆的，因为它的加密不仅与密钥和明文有关，而且和当前状态有关。
③分组密码算法的设计关键在于加解密算法，使明文和密文之间关联在密钥的控制下尽可能复杂；序列密码算法的设计关键在于密钥序列产生器，使生成的密钥序列具有不可预测性。

# 同步序列密码

如果密钥的产生独立于明文消息和密文消息，则此类序列密码称为同步序列密码。

在通信过程中，通信的双方必须保持精确的同步， 收方才能正确解密，如果失步收方将不能正确解密。例如，如果通信中丢失或增加了一个密文字符，则收方的解密将一直错误。

若 c2 丢失 (密文失步)，则 m1 后明文全错；若 c2 错误 (密文错误)，则仅 m2 明文出错。

注意：错误和失步是完全不同的两个概念。

特点：①对失步的敏感性，使我们能够容易检测插入、删除、重播等主动攻击。
②没有错误传播，当通信中某些密文字符产生了错误（0 错成 1 或 1 错成 0，不是插入和删除），只影响相应字符的解密，不影响其它字符。

# 自同步序列密码

如果密钥序列的产生是密钥及固定大小的以往密文位的函数，则这种序列密码被称为自同步序列密码或非同步序列密码。

特点：①有限错误传播：设密钥序列产生器具有 n 位存储，则一个符号的传输错误只影响到后面 n 符号的解密。
②自同步：只要接收方连续收到 n 个正确的密文符号，密钥序列产生器便会自动地恢复同步 。
③消除明文统计特性。

# 序列密码的的原理

特点：①加解密运算只是简单的模二加（异或）运算。
② 密码强度主要靠密钥流的随机性。

分析：①密钥流算法应该能产生随机性和不可预测性好的密钥序列；保持同步是序列密码在实际应用中的关键。
②密钥流都是按密钥生成算法生成，且要求通信双方能产生相同的密钥序列，所以不可能是真随机的 —— 伪随机序列。

# 伪随机序列

①极大的周期：随机序列是非周期的，而按任何算法产生的序列都是周期的，因此应要求密钥流具有尽可能大的周期。
②良好的统计特性：随机序列有均匀的游程分布；游程指序列中相同符号的连续段，其前后均为异种符号……0 111 0000 10…… ；有长为 3 的 1 游程、长为 4 的 0 游程、长为 1 的 1 游程。一般要求其在周期内满足：同样长度的 0 游程和 1 游程的个数相等，或近似相等。
③很高的线性复杂性：不能用级数较小的线性移位寄存器 LFSR 近似代替。
④用统计方法由密钥序列 k0k1k2...ki... 提取密钥生成器结构或种子密钥在计算上不可行。

# 寄存器

反馈移位寄存器 (feedback shift register,FSR) 是由 n 位的寄存器和反馈函数 (feedback function) 组成，n 位的寄存器中的初始值称为移位寄存器的初态。

线性移位寄存器 (Linear Shift Registor)
①n 级线性移位寄存器最多有 2n 个不同的状态。若其初始状态为零，则其后续状态恒为零。若其初始状态不为零，则其后续状态也不为零。因此，n 级线性移位寄存器的状态周期≤2n–1，其输出序列的周期≤2n–1。
②只要选择合适的连接多项式便可使线性移位寄存器的输出序列周期达到最大值 2n–1，并称此时的输出序列为最大长度线性移位寄存器输出序列，简称为 m 序列。
③仅当连接多项式 g (x) 为本原多项式时，其线性移位寄存器的输出序列为 m 序列。
设 g (x) 为 GF（2）上的多项式，使 g (x)|xp+1 的最小正整数 p 称为 g (x) 的周期。如果 g (x) 的次数为 n，且其周期为 2n-1，则称 g (x) 为本原多项式。已经证明，对于任意的正整数 n，至少存在一个 n 次本原多项式，而且存在有效的产生算法。

反馈移位寄存器 FSR 分为:
线性反馈移位寄存器 (LFSR)：主要采用。
非线性反馈移位寄存器 (NLFSR)：也可作为驱动器，但因为数学上分析困难很少被采用。
线性反馈移位寄存器 (liner feedback shift register,LFSR) 是一种特殊的 FSR，其反馈函数是移位寄存器中某些位的异或，参与运算的这些位叫做抽头位。
特征：n 级 LFSR 的有效状态为 2n-1 (全 0 状态除外，因全 0 状态的输出序列一直为全 0)，也即理论上能够产生周期为 2n-1 的伪随机序列。但要产生最大周期的输出序列，抽头位有一定要求。-> 本原多项式

# summary

①理论上说，无论采用多少阶，在有限次的运算下，一定可以得到正确的结果。本人亲测在 15 阶 - 20 阶线性反馈移位寄存器状态下都能成功跑出结果，这意味着序列密码抵抗不住穷举攻击。
②BUT，实际上，随着阶数的增加，计算机运行时间呈现指数增长趋势，只要指数足够大，计算机很难在段时间实现序列密码的破解，这说明，阶数越大安全性就越好，足够大的阶数足以保障序列密码安全性。
③由于本段程序采用普通容器进行数据的存取，并使用数组对 m 序列进行存储和输出，因此复杂度过高。由于时间紧迫，算法的优化只提出部分思路：
a. 使用数组来代替容器进行操作，由于本人能力有限，对数组的认知只停留在大一 C++ 课堂上，具体怎么实现尚未深入研究，据某佬透露，容器里包含了很多其他的东西，提高了算法复杂度，所以数组优于容器。
b. 可以用二进制容器 bitset 代替普通容器，对于这个容器的使用详见分组密码中的使用样板，bitset 只用来存储 0 和 1，可以进行移位操作，实践证明，它的效率确实比普通容器直接插入拿出操作快很多，如果时间足够，我倒是想尝试用此方法实现。
